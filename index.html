Marching Cubes

Source code on Github

An experiment with multithreaded marching cubes which degenerated, as should have been expected, into an LV-426 simulator.

Features

  • Multithreaded marching cubes tesselator, using an octree space subdivision to reduce the space to march
  • Simple OpenGL renderer, from first principles, no libraries or engines here
  • Several volume primitive types, up to an infinite paging terrain with automatic spatially stable random reebling system
  • Simple post processing stack, used for procedural sky and volumetric noise-based raymarched fog

Realtime multi-body tracking in VR

Work done at Vulcan, Inc to create realtime physical embodyment in VR, using multiple Intel RealSense depth cameras.

Features

  • 16 Intel RealSense depth cameras, with automatic extrinsics computed via computer-vision by observing a Steam Vive Tracker with an attached aruco.
  • Each depth camera had a dedicated Intel NUC running human body pose tracking software, and the poses were transformed to world space and sent to a pose aggregating server
  • The pose aggregating server spatio/temporally tracked humans in the room and computed the best pose skeletons for each through analysis of the 16 pose streams
  • All this was presented in real-time to users wearing a SteamVR headset, allowing users to see one another in the room and interact.

SGF / SRE

SimpleGameFramework & SimpleRenderingEngine: 3D Game framework written between 2007 and 2010. Written from ground-up ( as-in, I started with an empty project, and wrote it all myself) in C++ and some ObjectiveC, and with rendering in OpenGL/GLSL.

Features

  • Physics, using the OpenDynamicsEngine and Bullet, as pluggable modules
  • Shadow mapping for directional (using CSM), point and spot lights (using depth cubemaps), with efficient visibility culling of light volumes
  • Quirky materials framework
  • Voxel-based terrain, editable in real-time, with Lindstrom-Turk edge-collapse mesh optimization, uzing the CGAL library.
  • Threaded Python scripting for the physics system - mainly used to make autonomous robotics simulations that can query their environment and send commands to servos/motors/actuators/etc
  • Automatic environmental greebling details like grass, decals, etc using 3d perlin noise to keep distributions naturalistic
